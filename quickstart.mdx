# Quick Start Guide

This guide will walk you through parsing your first HOCON configuration with AHOCON.

## Basic Usage

<Steps>

<Step title="Install AHOCON">
  First, install the package using your preferred package manager:
  
  ```bash
  npm install ahocon
  ```
</Step>

<Step title="Import the parse function">
  Import the `parse` function from the AHOCON package:
  
  ```javascript
  import { parse } from 'ahocon';
  ```
  
  <Note>
    Use `const { parse } = require('ahocon');` if you're using CommonJS.
  </Note>
</Step>

<Step title="Parse your configuration">
  Call `parse()` with your HOCON configuration string:
  
  ```javascript
  const config = parse('example = true /* comment */');
  
  console.log(config); // { "example": true }
  ```
  
  The parser returns a JavaScript object representing your configuration.
</Step>

</Steps>

## TypeScript Usage

AHOCON is built in TypeScript and provides full type safety for your configurations.

<Steps>

<Step title="Define your configuration interface">
  Create a TypeScript interface that describes your configuration structure:
  
  ```typescript
  interface MyConfig {
    example: boolean;
  }
  ```
</Step>

<Step title="Use the type parameter">
  Pass your interface as a type parameter to the `parse` function:
  
  ```typescript
  import { parse } from 'ahocon';
  
  interface MyConfig {
    example: boolean;
  }
  
  const myConfig = parse<MyConfig>('example = true /* comment */');
  
  console.log(myConfig); // { "example": true }
  
  type TypeofMyConfig = typeof myConfig; // shows MyConfig
  ```
  
  The returned object will have the `MyConfig` type.
</Step>

</Steps>

<Tip>
  Type parameters provide autocomplete and type checking in your IDE, making it easier to work with complex configurations.
</Tip>

## Runtime Validation

AHOCON is a parser, not a schema validator. For runtime validation of user-provided configurations, use a validation library like [zod](https://www.npmjs.com/package/zod):

```typescript
import { parse } from 'ahocon';
import { z } from 'zod';

const schema = z
  .object({
    example: z.boolean(),
  })
  .strict(); // don't allow other keys

const myConfig = parse<z.infer<typeof schema>>('example = true');
const evilUserConfig = parse<z.infer<typeof schema>>('example = 123');

schema.parse(myConfig); // ✓ all good
schema.parse(evilUserConfig); // ✗ throws ZodError
```

<Note>
  TypeScript types are erased at runtime. Always validate untrusted input with a runtime validation library.
</Note>

## Handling Empty Input

If the input string is empty or contains only whitespace, `parse()` returns an empty object:

```typescript
const config = parse('');
console.log(config); // {}
```

## Working with Complex Configurations

HOCON supports nested objects, arrays, and various value types:

```javascript
const complexConfig = parse(`
  server {
    host = "localhost"
    port = 8080
    features = ["api", "admin"]
  }
  
  database {
    url = "postgresql://localhost/db"
    poolSize = 10
  }
`);

console.log(complexConfig.server.port); // 8080
console.log(complexConfig.database.poolSize); // 10
```
