## Overview

AHOCON allows you to extend parsing capabilities by writing custom functions. Functions enable dynamic computation during parsing, making your configurations more powerful and flexible.

<Warning>
  Custom functions are only available when using `ahocon/extended`. The standard parser does not support function parsing.
</Warning>

## Getting Started

<Steps>
  <Step title="Import the extended parser">
    Change your import from `ahocon` to `ahocon/extended`:
    
    ```javascript
    import { parse } from 'ahocon/extended';
    ```
  </Step>
  
  <Step title="Provide functions in parse options">
    Pass your custom functions via the `functions` option:
    
    ```javascript
    parse('example = true', {
      functions: {
        // your functions here
      },
    });
    ```
  </Step>
  
  <Step title="Use functions in your config">
    Functions are called with the `$` prefix:
    
    ```hocon
    result = $myFunction(arg1, arg2)
    ```
  </Step>
</Steps>

## CustomFunction Type

All custom functions must match the `AHOCON.CustomFunction` type signature:

```typescript
type CustomFunction = (params: {
  args: Array<unknown>;
  root: AHOCON.PickNode<'root'>;
  node: AHOCON.PickNode<'function'>;
  parseNode: (node: AHOCON.Node) => unknown;
}) => unknown;
```

### Parameters

- **`args`** - Array of parsed arguments passed to the function in the config
- **`root`** - The root node of the AST (Abstract Syntax Tree)
- **`node`** - The current function node being evaluated
- **`parseNode`** - Helper function to parse other nodes in the AST

<Note>
  Functions don't return values directly. Instead, set `node.evaluated` to assign the result.
</Note>

## Writing a Custom Function

Here's a complete example from the built-in math functions:

```typescript src/funcs/math.ts
const add: AHOCON.CustomFunction = ({ args, node }) => {
  let acc = 0;

  args.forEach((item) => {
    if (typeof item !== 'number')
      throw Error(
        `${node.name.value} can only work with numbers, got ${typeof item} (value = ${item})`
      );

    acc += item;
  });

  node.evaluated = acc;
};
```

### Best Practices

<Accordion title="Always validate argument types">
  Type assertions prevent runtime errors and provide helpful error messages:
  
  ```typescript
  if (typeof item !== 'number') {
    throw Error(`Expected number, got ${typeof item}`);
  }
  ```
</Accordion>

<Accordion title="Validate argument count">
  Ensure the correct number of arguments are provided:
  
  ```typescript
  if (args.length !== 2) {
    throw Error(`${node.name.value} needs exactly 2 arguments`);
  }
  ```
</Accordion>

<Accordion title="Set node.evaluated, don't return">
  Custom functions mutate the node instead of returning values:
  
  ```typescript
  node.evaluated = result; // ✓ Correct
  return result;           // ✗ Won't work
  ```
</Accordion>

<Accordion title="Keep operations lightweight">
  Functions run during parsing, so avoid heavy operations:
  
  - ✓ Simple calculations, string manipulation
  - ✓ Light validation and transformation
  - ✗ Network requests, heavy computation
  - ✗ File system operations (unless necessary)
</Accordion>

## Using Namespaces

Organize related functions into namespaces using nested objects:

```javascript
import { parse } from 'ahocon/extended';
import * as math from 'ahocon/funcs/math';

parse('result = $math.add(1, 2, 3)', {
  functions: {
    math, // namespace containing add, subtract, etc.
  },
});
```

In your AHOCON config:

```hocon
a = $math.add(1, 2, 3)        # Result: 6
b = $math.multiply(2, 3, 4)   # Result: 24
```

### Without Namespaces

You can also import functions directly:

```javascript
import { add } from 'ahocon/funcs/math';

parse('result = $add(1, 2, 3)', {
  functions: { add },
});
```

<Note>
  While namespace imports (`import *`) are generally discouraged for tree-shaking reasons, they're acceptable when you're using all functions in the namespace.
</Note>

## Advanced: Using the AST

The `root` parameter provides access to the entire AST, enabling advanced use cases like referencing other config values.

Example from the `ref` function:

```typescript src/funcs/ref.ts
const ref: AHOCON.CustomFunction = ({ args, node, root, parseNode }) => {
  if (args.length !== 1) throw Error('ref failed, too many arguments');

  let partials = (args[0] as string).split('.');
  let reference: AHOCON.Node | null | undefined = root as AHOCON.Node;

  // relative path (starts with .)
  if (partials[0] === '') {
    partials.shift();
    reference = node.parent as typeof reference;
  }

  partials.forEach((partial) => {
    if (!reference) return;

    if (reference.type === 'root') reference = reference.entry;

    if (reference.type === 'object')
      reference = reference.props.find((item) => item.key.value === partial)?.value;
    else if (reference.type === 'array')
      reference = reference.values[partial as unknown as number];
  });

  if (reference.type === 'string' || reference.type === 'raw') 
    node.evaluated = reference.value;
  else if (reference.type === 'array' || reference.type === 'object' || reference.type === 'function')
    node.evaluated = parseNode(reference);
  else if (reference.type === 'root') 
    node.evaluated = parseNode(reference.entry);
};
```

<Warning>
  When traversing the AST, be careful to avoid creating infinite loops. AHOCON does not protect against circular references in custom functions.
  
  ```hocon
  # This will cause an infinite loop!
  a = $ref(b)
  b = $ref(a)
  ```
</Warning>

## Example: Custom Converter Function

Here's a real-world example of a memory size converter:

```typescript src/funcs/convert.ts
const memoryTable: Record<string, number> = {
  byte: 1,
  kilobyte: Math.pow(10, 3),
  megabyte: Math.pow(10, 6),
  gigabyte: Math.pow(10, 9),
  kb: Math.pow(10, 3),
  mb: Math.pow(10, 6),
  gb: Math.pow(10, 9),
};

const memorysize: AHOCON.CustomFunction = ({ args, node }) => {
  if (args.length !== 2) 
    throw Error(`${node.name.value} needs exactly 2 arguments (from and to)`);
  
  if (typeof args[0] !== 'string' || typeof args[1] !== 'string')
    throw Error(`${node.name.value} needs the arguments to be strings`);

  const value1 = parseFloat(args[0]);
  const unit1 = args[0].match(/[a-z]+/)?.[0]?.toLowerCase();
  const unit2 = args[1].match(/[a-z]+/)?.[0]?.toLowerCase();

  if (!unit1 || !unit2)
    throw Error(
      `${node.name.value} arguments need a unit to convert from and a unit to convert to`
    );

  if (!memoryTable[unit1]) throw Error(`invalid unit ${unit1}`);
  if (!memoryTable[unit2]) throw Error(`invalid unit ${unit2}`);

  node.evaluated = `${(memoryTable[unit1] * value1) / memoryTable[unit2]}${unit2}`;
};
```

Usage:

```hocon
memory = $convert.memorysize("512mb", "gb")  # Result: "0.512gb"
```
