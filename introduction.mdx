# Welcome to AHOCON

AHOCON (short for Another HOCON) is a TypeScript implementation of the [HOCON grammar](https://github.com/lightbend/config/blob/master/HOCON.md) that provides a simple, human-friendly configuration format with powerful features.

## What is HOCON?

HOCON (Human-Optimized Config Object Notation) is a superset of JSON that provides a more flexible and readable way to write configuration files. AHOCON parses HOCON syntax and converts it to standard JSON objects.

## Quick Example

Transform HOCON configuration into typed JavaScript objects:

```typescript
import { parse } from 'ahocon';

interface Config {
  example: boolean;
}

const config = parse<Config>('example = true /* comment */');

console.log(config); // { "example": true }
```

## Use Cases

- Application configuration files
- Build tool configurations
- Environment-specific settings
- Complex nested configuration structures
- Any scenario where JSON feels too rigid

<Note>
  AHOCON is a parser, not a schema validator. For runtime validation, consider using [zod](https://www.npmjs.com/package/zod) or similar libraries.
</Note>
